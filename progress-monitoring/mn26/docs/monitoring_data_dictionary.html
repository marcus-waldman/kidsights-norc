MN26 Monitoring Data Dictionary

Variable definitions for generate_monitoring_report() output

Published

April 1, 2026

Data dictionary for the 6 data frames returned by generate_monitoring_report().

Each row in every data frame represents one REDCap record, identified by pid + record_id.

Note

Variables with a _norc suffix use MN26 NORC value codes that differ from the NE25 pipeline.

1 $screener_status

Tracks whether eligibility has been determined for each participant.

1.1 Columns

Column Source Variable Type Description Values
pid pid (from credentials CSV) integer REDCap project ID Project-specific integer
record_id record_id numeric REDCap record ID Unique per record
screener_complete Derived logical Whether eligibility is known (non-missing) TRUE, FALSE
screener_status Derived from screener_complete character Label for screener completion "Complete", "Incomplete"

1.2 Derivation

  • screener_complete: TRUE if all 4 eligibility criteria have been answered (non-NA)
  • screener_status: "Complete" when screener_complete == TRUE, otherwise "Incomplete"

2 $eligibility

Four eligibility criteria and overall eligibility determination.

2.1 Columns

Column Source Variable Type Description Values
pid pid integer REDCap project ID Project-specific integer
record_id record_id numeric REDCap record ID Unique per record
parent_age_eligible eq003 logical Parent age >= 19 TRUE, FALSE
child_age_eligible age_in_days_n logical Child age 0–5 years TRUE, FALSE
primary_caregiver_eligible eq002 logical Respondent is primary caregiver TRUE, FALSE
state_eligible mn_eqstate logical Lives in Minnesota TRUE, FALSE
eligible Derived logical Overall eligibility (all 4 criteria) TRUE, FALSE

2.2 Variable Mappings

parent_age_eligible from eq003 (yesno)

eq003 == 1 AND not missing \(\rightarrow\) TRUE; otherwise \(\rightarrow\) FALSE

child_age_eligible from age_in_days_n (calculated field)

age_in_days_n <= 1825 (5 years) AND not missing \(\rightarrow\) TRUE; otherwise \(\rightarrow\) FALSE

primary_caregiver_eligible from eq002 (yesno)

eq002 == 1 AND not missing \(\rightarrow\) TRUE; otherwise \(\rightarrow\) FALSE

state_eligible from mn_eqstate (yesno)

mn_eqstate == 1 AND not missing \(\rightarrow\) TRUE; otherwise \(\rightarrow\) FALSE

Tip

NE25 used eqstate (Nebraska residence); MN26 uses mn_eqstate (Minnesota residence).

eligible

parent_age_eligible & child_age_eligible & primary_caregiver_eligible & state_eligible

3 $survey_completion

Module-by-module completion tracking.

3.1 Columns

Column Source Variable Type Description Values
pid pid integer REDCap project ID Project-specific integer
record_id record_id numeric REDCap record ID Unique per record
modules_complete Derived integer Count of completed modules 0 to N
pct_complete Derived numeric Percentage of modules complete 0 to 100
last_module_complete Derived character Name of last completed module Module name or "None"
survey_complete Derived logical All required modules complete TRUE, FALSE
survey_status Derived character Label for survey completion "Complete", "Incomplete"

3.2 Derivation

  • A module is “complete” when its REDCap instrument status = 2
  • survey_complete: TRUE when all required modules have status = 2
Warning

pct_complete is currently incorrect. The required module list is carried over from NE25 and does not match MN26. Specifically: Module 4 (Home Learning Environment) does not exist in MN26 and is silently dropped, nsch_questions is new in MN26 but not included, and child 2 modules (module_6_*_2, nsch_questions_2) are unaccounted for. See GitHub Issue #1 for the full list of questions that need resolving before this data frame is reliable.

4 $child_demographics

Age, sex, and race/ethnicity for child 1 and child 2.

4.1 Columns

Column Source Variable Type Description Values
pid pid integer REDCap project ID Project-specific integer
record_id record_id numeric REDCap record ID Unique per record
age_years age_in_days_n numeric Child 1 age in years 0–5 (continuous)
sex_norc cqr009 character Child 1 sex "Female", "Male"
race_norc cqr010b___* character Child 1 race See mapping below
hisp cqr011 character Child 1 Hispanic/Latino ethnicity "Hispanic", "non-Hisp."
raceG_norc Derived character Child 1 combined race/ethnicity See mapping below
years_old_c2 age_in_days_c2_n numeric Child 2 age in years 0–5, NA if no child 2
sex_c2_norc cqr009_c2 character Child 2 sex "Female", "Male", NA if no child 2
race_c2_norc cqr010_c2b___* character Child 2 race Same categories as child 1
hisp_c2 cqr011_c2 character Child 2 Hispanic/Latino ethnicity "Hispanic", "non-Hisp."
raceG_c2_norc Derived character Child 2 combined race/ethnicity Same categories as child 1

4.2 Variable Mappings

4.2.1 Age

age_years = age_in_days_n / 365.25

Tip

NE25 used age_in_days; MN26 uses age_in_days_n (child 1) and age_in_days_c2_n (child 2).

4.2.2 Sex

sex_norc from cqr009 (radio). Child 2 sex_c2_norc from cqr009_c2 uses the same codes.

REDCap Code Label
1 Female
0 Male
Important

Codes are swapped from NE25 (NE25: 0=Female, 1=Male).

4.2.3 Race

race_norc from cqr010b (checkbox, check all that apply). Child 2 race_c2_norc from cqr010_c2b___* uses the same codes.

REDCap Code Checkbox Field Label
100 cqr010b___100 American Indian or Alaska Native
101 cqr010b___101 Asian
102 cqr010b___102 Black or African American
103 cqr010b___103 Native Hawaiian or Other Pacific Islander
104 cqr010b___104 White
105 cqr010b___105 Other
  • If multiple checkboxes selected \(\rightarrow\) "Two or More"
  • If no checkboxes selected \(\rightarrow\) NA
Tip

NE25 used cqr010___* with 15 granular categories; MN26 uses cqr010b___* with 6 pre-collapsed categories.

4.2.4 Ethnicity

hisp from cqr011 (radio). Child 2 hisp_c2 from cqr011_c2 uses the same codes.

REDCap Code Label
0 non-Hisp.
1 Hispanic

4.2.5 Combined Race/Ethnicity

raceG_norc derived from race_norc + hisp:

  • If hisp == "Hispanic" \(\rightarrow\) "Hispanic"
  • Otherwise \(\rightarrow\) "{race_norc}, non-Hisp." (e.g., "White, non-Hisp.")

5.2.1 Gender

gender (a1_gender_norc) from mn2 (radio):

REDCap Code Label
0 Female
1 Male

Tip

NE25 used cqr002 (binary sex: 0=Female, 1=Male); MN26 uses mn2 (gender with Non-binary option).

  • If no checkboxes selected \(\rightarrow\) NA

REDCap Code Label
