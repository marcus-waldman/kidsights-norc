MN26 Monitoring Report Function Guide

Input Requirements & Output Specifications for NORC

Published

February 6, 2026

Overview

The generate_monitoring_report() function is a self-contained R function that pulls raw data from REDCap, transforms key demographic variables, and returns structured data frames for monitoring MN26 recruitment progress. This document specifies what NORC needs to provide (inputs) and what the function returns (outputs).

Current Status: The script uses NE25 (Nebraska 2025) data as a template. Variable names will be updated once MN26 data collection begins.

Multi-Child Households

MN26 allows up to 2 children per household. The current script assumes 1 child per record (NE25 structure). Once the MN26 REDCap data dictionary is finalized, the child demographics and child age eligibility logic will need to be updated so that each child is tracked separately. The exact modification depends on how REDCap stores the second child’s data (repeating instruments, separate fields, etc.).

This affects:

  • Child demographics — will return up to 2 rows per household instead of 1
  • Eligibility — child age (0–5 years) will need to be checked per child

What You Need (Inputs)

R Packages

Install the following R packages before first use:

install.packages(c("dplyr", "tidyr", "REDCapR", "httr"))

API Credentials CSV File

The function requires a single CSV file containing REDCap API credentials.

CSV columns:

Column Description Example
project Descriptive project name mn26_main_survey
pid REDCap project ID number 7679
api_code REDCap API token (32-character string) A1B2C3D4E5F6G7H8...

Example CSV file (mn26_redcap_api.csv):

project,pid,api_code
mn26_main_survey,7679,A1B2C3D4E5F6G7H8I9J0K1L2M3N4O5P6
mn26_follow_up,8014,Q7R8S9T0U1V2W3X4Y5Z6A7B8C9D0E1F2

Store this file in a secure location outside the repository (e.g., C:/Users/YOUR_USERNAME/my-APIs/mn26_redcap_api.csv). Never commit this file to version control.

How to Run It

Basic Usage

# Step 1: Load the script
source("monitoring_report.R")

# Step 2: Run the function (pass your CSV path)
monitoring_data <- generate_monitoring_report(
  csv_path = "C:/Users/YOUR_USERNAME/my-APIs/mn26_redcap_api.csv"
)

Optional: Custom REDCap URL

If MN26 uses a different REDCap instance than the default:

monitoring_data <- generate_monitoring_report(
  csv_path = "C:/Users/YOUR_USERNAME/my-APIs/mn26_redcap_api.csv",
  redcap_url = "https://redcap.umn.edu/api/"
)

Function Parameters

Parameter Required Default Description
csv_path Yes – Path to API credentials CSV
redcap_url No https://unmcredcap.unmc.edu/redcap/api/ REDCap API endpoint URL

What You Get Back (Outputs)

The function returns a named list with 5 data frames. Access each one using $ notation (e.g., monitoring_data$eligibility).

1. $screener_status — Screener Completion

Tracks whether eligibility has been determined for each participant.

Columns:

Column Type Description
pid character Participant ID
record_id integer REDCap record ID
screener_complete logical TRUE if eligibility is known
screener_status character "Complete" or "Incomplete"

Hypothetical output (head(monitoring_data$screener_status)):

pid record_id screener_complete screener_status
MN26-0001 1 TRUE Complete
MN26-0002 2 TRUE Complete
MN26-0003 3 FALSE Incomplete
MN26-0004 4 TRUE Complete
MN26-0005 5 TRUE Complete
MN26-0006 6 FALSE Incomplete

Example summary:

table(monitoring_data$screener_status$screener_status)
#
#   Complete Incomplete
#      1,842        658

3. $survey_completion — Survey Module Completion

Tracks progress through the required survey modules (Modules 2, 3, 4, 5, 6, 7, and 9). Includes the number and percentage of modules completed, and which module was the last one finished.

Column Type Description

record_id integer REDCap record ID

Putting It All Together

A typical monitoring session might look like this:

# Load and run
source("monitoring_report.R")
monitoring_data <- generate_monitoring_report(
  csv_path = "C:/Users/YOUR_USERNAME/my-APIs/mn26_redcap_api.csv"
)

# Quick recruitment snapshot
cat("Total screened:", nrow(monitoring_data$screener_status), "\n")
cat("Screener complete:",
    sum(monitoring_data$screener_status$screener_complete), "\n")
cat("Eligible:",
    sum(monitoring_data$eligibility$eligible, na.rm = TRUE), "\n")
cat("Survey complete:",
    sum(monitoring_data$survey_completion$survey_complete, na.rm = TRUE), "\n")

# Which eligibility criterion is failing most?
elig <- monitoring_data$eligibility
cat("\nEligibility pass rates:\n")
cat("  Parent age 19+:",
    mean(elig$parent_age_eligible, na.rm = TRUE) * 100, "%\n")
cat("  Child age 0-5:",
    mean(elig$child_age_eligible, na.rm = TRUE) * 100, "%\n")
cat("  Primary caregiver:",
    mean(elig$primary_caregiver_eligible, na.rm = TRUE) * 100, "%\n")
cat("  Lives in Minnesota:",
    mean(elig$state_eligible, na.rm = TRUE) * 100, "%\n")

# Demographic balance check
table(monitoring_data$child_demographics$sex)
table(monitoring_data$parent_demographics$race_ethnicity)
table(monitoring_data$parent_demographics$education)

Geography

The monitoring function does not currently include geographic data. However, the mailing address where each participant’s invitation was sent can be identified by linking pid + record_id to a separate address dataset (not included in the REDCap API pull).

Future versions of this script may add geocoding capabilities to convert addresses to latitude/longitude and derive geographic classifications (county, census tract, PUMA, urban/rural) using the US Census Bureau geocoder. This would enable monitoring of geographic coverage across Minnesota during recruitment.
